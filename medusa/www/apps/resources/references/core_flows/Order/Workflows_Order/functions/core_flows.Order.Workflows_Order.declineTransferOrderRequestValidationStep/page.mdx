# declineTransferOrderRequestValidationStep - Medusa Core Workflows Reference

This documentation provides a reference to the `declineTransferOrderRequestValidationStep`. It belongs to the `@medusajs/medusa/core-flows` package.

This step validates that a requested order transfer can be declineed.
If the provided token doesn't match the token of the transfer request,
the step throws an error.

:::note

You can retrieve an order and order change details using [Query](https://docs.medusajs.com/learn/fundamentals/module-links/query),
or [useQueryGraphStep](https://docs.medusajs.com/resources/references/medusa-workflows/steps/useQueryGraphStep).

:::

## Example

```ts
const data = declineTransferOrderRequestValidationStep({
  order: {
    id: "order_123",
    // other order details...
  },
  orderChange: {
    id: "order_change_123",
    // other order change details...
  },
  input: {
    token: "token_123",
    order_id: "order_123",
  }
})
```
