# receiveItemReturnRequestValidationStep - Medusa Core Workflows Reference

This documentation provides a reference to the `receiveItemReturnRequestValidationStep`. It belongs to the `@medusajs/medusa/core-flows` package.

This step validates that a return's items can be marked as received.
If the order or return is canceled, the order change is not active,
or the items do not exist in the return, the step will throw an error.

:::note

You can retrieve an order, return, and order change details using [Query](https://docs.medusajs.com/learn/fundamentals/module-links/query),
or [useQueryGraphStep](https://docs.medusajs.com/resources/references/medusa-workflows/steps/useQueryGraphStep).

:::

## Example

```ts
const data = receiveItemReturnRequestValidationStep({
  order: {
    id: "order_123",
    // other order details...
  },
  orderChange: {
    id: "orch_123",
    // other order change details...
  },
  orderReturn: {
    id: "return_123",
    // other return details...
  },
  items: [
    {
      id: "orli_123",
      quantity: 1,
    }
  ]
})
```
