# listAccountTransactions - Store Credit Module Reference

This documentation provides a reference to the `listAccountTransactions` method. This belongs to the Store Credit Module.

<Note>

You should only use this methods when implementing complex customizations. For common cases, check out [available workflows instead](/medusa-workflows-reference).

</Note>

This method retrieves a list of account transactions based on the provided filters.

## Example

To retrieve a list of account transactions by ID:

```ts
const transactions = await storeCreditModuleService.listAccountTransactions({
  id: ["txn_123", "txn_321"],
})
```

To specify relations that should be retrieved within the account transactions:

:::note

You can only retrieve data models defined in the same module. To retrieve linked data models
from other modules, use [Query](https://docs.medusajs.com/learn/fundamentals/module-links/query) instead.

:::

```ts
const transactions = await storeCreditModuleService.listAccountTransactions({
  id: ["txn_123", "txn_321"],
}, {
  relations: ["account"],
})
```

By default, only the first `15` records are retrieved. You can control pagination by specifying the `skip` and `take` properties of the `config` parameter:

```ts
const transactions = await storeCreditModuleService.listAccountTransactions(
  { id: ["txn_123", "txn_321"] },
  { take: 20, skip: 2 }
)
```
