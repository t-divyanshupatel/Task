In this guide, you'll learn about outgoing (egress) and incoming (ingress) IP addresses for your Medusa project deployed on Cloud.

## Outgoing or Egress IP Addresses

When your Medusa backend makes requests to external services, such as calling a payment provider's API, sending webhooks to your systems, or connecting to external APIs, those requests originate from fixed IP addresses.

If you need to whitelist Cloud's outgoing IP addresses in your firewall or allowlist them in external services, you can use the following static IP addresses.

## Incoming or Ingress IP Addresses

Incoming (ingress) IP addresses are dynamic and change over time. Only outgoing (egress) IP addresses are static and suitable for allowlisting.

If you need to access the incoming IP address of your Medusa project, contact support for help with setting up a proxy for your Medusa project.
