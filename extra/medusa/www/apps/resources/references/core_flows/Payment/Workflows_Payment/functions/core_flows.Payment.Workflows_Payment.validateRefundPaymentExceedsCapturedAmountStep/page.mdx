# validateRefundPaymentExceedsCapturedAmountStep - Medusa Core Workflows Reference

This documentation provides a reference to the `validateRefundPaymentExceedsCapturedAmountStep`. It belongs to the `@medusajs/medusa/core-flows` package.

This step validates that the refund amount does not exceed the captured amount for a payment.
The validation accounts for currency decimal precision to handle rounding issues.

## Example

```ts
validateRefundPaymentExceedsCapturedAmountStep({
  payment: paymentData,
  refundAmount: 100.50
})
```
